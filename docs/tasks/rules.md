# Copilot Task Execution Rules

## Core rules
- Only ONE task can be active at a time
- Never implement more than one feature per task
- Always read active_task.md before coding
- Always update done.md after completion

## Execution flow
1. Read context.md
2. Read active_task.md
3. Implement ONLY current task scope
4. Stop immediately after completion
5. Wait for next instruction

## Forbidden
- Do not refactor unrelated modules
- Do not implement future tasks
- Do not redesign architecture
